# **App Name**: VeriTix

## Core Features:

- Public Event Discovery: Users can browse all available events, view detailed information including name, description, date, time, venue, and ticket price.
- Interactive Seat Selection: A visual, real-time updated seat map allowing users to select their preferred seats before purchasing tickets, ensuring no double-booking.
- Streamlined Ticket Purchase: A simplified multi-step checkout process with simulated payment integration to allow users to easily buy tickets without direct blockchain interaction.
- Personalized Ticket Dashboard: A dedicated user section where purchased tickets are displayed. Users can view event details and generate a downloadable QR code containing ticket ID, event ID, and verification hash.
- Event & Ticket Management Dashboard (Admin): A secure, authenticated administration portal for creating new events with details like capacity, price, date, and time. Admins can also view all active events and monitor sold tickets.
- Secure Background NFT Minting: Automated server-side process that securely interacts with the ERC-721 smart contract to mint NFT tickets to the user's wallet upon a successful ticket purchase, abstracting blockchain complexities from the user.
- AI-Powered Event Description Assistant (Admin): A tool within the admin dashboard that uses AI to assist in generating or refining engaging event descriptions based on keywords, themes, or event type, enhancing content creation efficiency.

## Style Guidelines:

- Color Anchor: Modernity, efficiency, security. Scheme: Dark. Primary: A vibrant, clear blue representing technology and trust (#52B3ED). Background: A heavily desaturated, very dark blue (#15191C). Accent: A brighter, contrasting purple-blue for highlights and CTAs (#BF8CE0).
- Headline font: 'Space Grotesk' (sans-serif) for a modern, techy aesthetic. Body font: 'Inter' (sans-serif) for neutral readability and excellent clarity across various screen sizes.
- Use clean, minimalist line icons that align with a modern and intuitive user interface, avoiding overly complex or skeuomorphic designs.
- Employ a responsive, grid-based layout for optimal content organization and readability across all devices, ensuring accessibility and ease of navigation.
- Implement subtle, smooth transitions and micro-interactions (e.g., button hovers, content loading) to provide visual feedback and enhance user experience without being distracting.
